## 一、基础符号

### 1.1 变量命名惯例

在 AI 领域，变量名约定俗成的惯例：

| 符号 | 含义 | 为什么用这个字母 |
|------|------|---------------|
| x | 输入数据 | 习惯用法 |
| y | 输出/标签 | y 跟在 x 后面 |
| W | 权重矩阵 | Weight |
| b | 偏置 | bias |
| L | 损失函数 | Loss |
| η (eta) | 学习率 | 希腊字母传统 |
| Q, K, V | 查询、键、值 | Query, Key, Value |
| N | 序列长度 | Number（数量） |
| d | 维度 | dimension |
| S | 注意力分数 | Score |
| P | 注意力概率 | Probability |
| O | 输出 | Output |
| M | SRAM 大小 | Memory |

### 1.2 大写 vs 小写的区别

这个非常重要！

```
大写字母 → 矩阵（一个表格/二维数组）
小写字母 → 标量（一个数字）或向量（一行/一列数字）

W = 权重矩阵（比如 1024×1024 的表格）
w = 某一个权重值（比如 0.37）

X = 整个输入矩阵
x = 一个输入样本或一个元素
```

## 二、下标与上标

### 2.1 下标（subscript）

下标写在右下方，用来表示 **位置、索引、编号**。

| 写法 | 读法 | 含义 |
|------|------|------|
| aᵢ | "a sub i" | 向量 a 的第 i 个元素 |
| Aᵢⱼ | "A sub i j" | 矩阵 A 的第 i 行、第 j 列的元素 |
| W₁, W₂ | "W one, W two" | 第 1 层、第 2 层的权重 |

```
矩阵 A 的样子：

         第1列  第2列  第3列
第1行 →  A₁₁   A₁₂   A₁₃
第2行 →  A₂₁   A₂₂   A₂₃
第3行 →  A₃₁   A₃₂   A₃₃

A₂₃ = 第2行、第3列的那个数
```

### 2.2 上标（superscript）

上标写在右上方，通常表示 **幂次、层数、转置**。

| 写法 | 含义 | 备注 |
|------|------|------|
| x² | x 的平方 | 数学里的幂 |
| xⁿ | x 的 n 次方 | |
| eˣ | 自然指数 | e ≈ 2.718 |
| W⁽ˡ⁾ | 第 l 层的权重 | 加括号区分"层数"和"幂次" |
| Aᵀ | A 的转置 | 见下文详解 |
| A⁻¹ | A 的逆矩阵 | |

> **如何区分 W²（平方）和 W⁽²⁾（第2层）？**
> 加了小括号 W⁽²⁾ 就是"第 2 层"，没括号 W² 就是"平方"。

## 三、矩阵相关符号

### 3.1 转置 Aᵀ

转置 = **行变列，列变行**（沿对角线翻转）。

```
原矩阵 A (2×3):          转置 Aᵀ (3×2):

┌───┬───┬───┐           ┌───┬───┐
│ 1 │ 2 │ 3 │           │ 1 │ 4 │
├───┼───┼───┤    →      ├───┼───┤
│ 4 │ 5 │ 6 │           │ 2 │ 5 │
└───┴───┴───┘           ├───┼───┤
                        │ 3 │ 6 │
                        └───┴───┘

规则：A 的第 i 行第 j 列 → Aᵀ 的第 j 行第 i 列
即：(Aᵀ)ᵢⱼ = Aⱼᵢ
```

**在 Attention 中为什么频繁出现转置？** 因为 QKᵀ 这个操作的含义是让 Q 的每一行和 K 的每一行做点积（衡量相似度）。如果不转置，矩阵乘法的维度对不上。

### 3.2 矩阵乘法

两个矩阵相乘 C = AB，规则：**左矩阵的一行 × 右矩阵的一列**

> A (m × **n**) · B (**n** × p) = C (m × p)
>
> 中间的维度 n 必须相同，否则无法相乘！

```
A (2×3) × B (3×2) = C (2×2)

   B =  ┌───┬───┐
        │ 1 │ 4 │
        │ 2 │ 5 │
        │ 3 │ 6 │
        └───┴───┘
                    C = ┌────┬────┐
A = ┌───┬───┬───┐       │ 14 │ 32 │  ← 1×1+2×2+3×3=14, 1×4+2×5+3×6=32
    │ 1 │ 2 │ 3 │       ├────┼────┤
    │ 4 │ 5 │ 6 │       │ 32 │ 77 │  ← 4×1+5×2+6×3=32, 4×4+5×5+6×6=77
    └───┴───┴───┘       └────┴────┘

C₁₁ = A的第1行 · B的第1列 = 1×1 + 2×2 + 3×3 = 14
```

### 3.3 矩阵维度标注 ℝᵐˣⁿ

| 写法 | 含义 |
|------|------|
| ℝ | 实数集合（Real numbers） |
| ℝⁿ | n 维向量（n 个实数组成的一列） |
| ℝᵐˣⁿ | m 行 n 列的矩阵 |
| Q ∈ ℝᴺˣᵈ | "Q 是一个 N 行 d 列的实数矩阵" |

在 Attention 论文中你会经常看到：

> Q, K, V ∈ ℝᴺˣᵈ，S, P ∈ ℝᴺˣᴺ

这只是在告诉你每个矩阵的大小。

## 四、求和符号 Σ

### 4.1 基本用法

Σ 读作 "sigma"（西格玛），表示 **把一堆东西加起来**。

```
 n
 Σ  aᵢ  =  a₁ + a₂ + a₃ + ... + aₙ
i=1

含义：i 从 1 开始，到 n 结束，把所有 aᵢ 加起来
```

### 4.2 在 Attention 中的例子

**Softmax 的分母：**

```
  N
  Σ  e^(Sᵢₖ)  =  e^(Sᵢ₁) + e^(Sᵢ₂) + ... + e^(SᵢN)
 k=1

含义：对第 i 行的所有列 k，把 e^(Sᵢₖ) 全部加起来。
```

**矩阵乘法的定义：**

```
        n
Cᵢⱼ =  Σ  Aᵢₖ · Bₖⱼ
       k=1

含义：C 的第 i 行第 j 列 = A 的第 i 行逐个乘 B 的第 j 列，然后全部加起来。
```

### 4.3 简写形式

论文中经常省略上下界：

| 写法 | 完整形式 | 含义 |
|------|---------|------|
| Σₖ | Σₖ₌₁ᴺ | 对所有 k 求和（范围从上下文推断） |
| Σᵢⱼ | Σᵢ Σⱼ | 对所有 i 和 j 双重求和 |
| Σₖ Pₖ | P₁ + P₂ + ... + Pₙ | 把向量 P 的所有元素加起来 |

## 五、微积分 / 导数符号

这是反向传播的核心。

### 5.1 普通导数 dy/dx

```
dy/dx  读作 "dy dx"，意思是：x 变化一点点时，y 变化多少

例子：
  y = x²
  dy/dx = 2x

  意思：如果 x = 3，那 dy/dx = 6
        即 x 增大 0.001，y 大约增大 6 × 0.001 = 0.006

  验证：3² = 9，3.001² = 9.006001 ✓ 差不多增大了 0.006
```

### 5.2 偏导数 ∂f/∂x

当函数有 **多个输入变量** 时，对其中一个求导（其他的当常数），用 ∂（读作 "partial"）。

```
f(x, y) = x² + 3xy

∂f/∂x = 2x + 3y    （把 y 当常数，只对 x 求导）
∂f/∂y = 3x          （把 x 当常数，只对 y 求导）
```

### 5.3 梯度的简写：dX

在深度学习论文中，为了简洁，常常这样简写：

| 简写 | 完整写法 | 含义 |
|------|---------|------|
| dX 或 ∇ₓL | ∂L/∂X | 损失 L 对 X 的梯度 |
| dW | ∂L/∂W | 损失 L 对 W 的梯度 |
| dO | ∂L/∂O | 损失对 Attention 输出 O 的梯度 |
| dS | ∂L/∂S | 损失对注意力分数 S 的梯度 |

> **重点理解：dX 和 X 形状完全相同！**
>
> 如果 X 是 N×d 的矩阵，那 dX 也是 N×d 的矩阵。
> dX 中的每个元素 dXᵢⱼ 表示"如果把 Xᵢⱼ 稍微调大一点，损失 L 会变化多少"。

### 5.4 链式法则的符号

```
∂L/∂x = ∂L/∂y · ∂y/∂x

直觉理解：

x 变化 → 导致 y 变化 → 导致 L 变化

L 对 x 的       L 对 y 的     y 对 x 的
总变化率    =    变化率    ×    变化率

就像汇率换算：
  人民币→美元→欧元
  人民币→欧元的汇率 = 人民币→美元的汇率 × 美元→欧元的汇率
```

## 六、特殊运算符号

### 6.1 逐元素乘法 ⊙（Hadamard 积）

**注意区分**：矩阵乘法 vs 逐元素乘法

```
矩阵乘法 (·)：                    逐元素乘法 (⊙)：
行 × 列，求和                      对应位置直接相乘

┌───┬───┐   ┌───┬───┐           ┌───┬───┐   ┌───┬───┐
│ 1 │ 2 │ × │ 5 │ 6 │           │ 1 │ 2 │ ⊙│ 5 │ 6 │
│ 3 │ 4 │   │ 7 │ 8 │           │ 3 │ 4 │   │ 7 │ 8 │
└───┴───┘   └───┴───┘           └───┴───┘   └───┴───┘

= ┌──────────┬──────────┐       = ┌────┬────┐
  │1×5+2×7=19│1×6+2×8=22│         │  5 │ 12 │
  │3×5+4×7=43│3×6+4×8=50│         │ 21 │ 32 │
  └──────────┴──────────┘         └────┴────┘
```

**在 Attention 反向传播中出现的地方：**

```
dS = P ⊙ (dP - rowsum(P ⊙ dP))

这里 P ⊙ dP 就是 P 和 dP 对应位置相乘。
```

### 6.2 Softmax

```
                    e 的 Sᵢⱼ 次方
softmax(Sᵢ)ⱼ = ─────────────────────────────────
                 e^(Sᵢ₁) + e^(Sᵢ₂) + ... + e^(SᵢN)

作用：把一行数字变成"概率分布"
  输入：[2.0, 1.0, 0.1]（任意实数）
  输出：[0.66, 0.24, 0.10]（全部为正，加起来=1）
```

### 6.3 argmin 和 argmax

| 写法 | 含义 |
|------|------|
| min f(x) | f(x) 的 **最小值** 是多少 |
| argmin f(x) | 让 f(x) 取到最小值的 **x 是什么** |

```
f(x) = (x-3)²

min f(x) = 0          ← 最小值是 0
argmin f(x) = 3       ← x=3 时取到最小值
```

## 七、复杂度符号 O(·)

在 FlashAttention 论文中频繁出现。

### 7.1 大 O 表示法

O(·) 表示 **数量级**，忽略常数因子，只关心 **增长趋势**。

| 写法 | 含义 | 举例（N = 1000） |
|------|------|---|
| O(1) | 常数，和 N 无关 | 1 次操作 |
| O(N) | 线性增长 | ~1,000 次操作 |
| O(N log N) | 稍快于线性 | ~10,000 次操作 |
| O(N²) | 平方增长 | ~1,000,000 次操作 |
| O(N²d) | 和 N² 及 d 都相关 | ~64,000,000 次（若 d=64） |

```
为什么 Attention 慢？

序列长度 N = 4096 时：
  N×d 矩阵：4096 × 64 = 262,144 个数      ← 还行
  N×N 矩阵：4096 × 4096 = 16,777,216 个数  ← 大了 64 倍！

序列长度翻倍 N = 8192 时：
  N×d 矩阵：翻 2 倍 → 524,288
  N×N 矩阵：翻 4 倍 → 67,108,864           ← 平方增长！
```

### 7.2 在 FlashAttention 中的具体含义

```
Standard Attention HBM 访问量 = O(N²)
FlashAttention HBM 访问量     = O(N²d / M)

其中 M 是 SRAM 的大小。
由于 M 是一个较大的常数（比如几 MB），所以实际访问量比 O(N²) 小很多。
```

## 九、公式阅读实战

用学到的所有符号，来读懂 FlashAttention 论文中的关键公式。

### 公式 1：Attention 前向传播

```
O = softmax(QKᵀ / √d) V
```

逐个拆解：

```
Q                → Query 矩阵 (N×d)
K                → Key 矩阵 (N×d)
Kᵀ               → K 的转置 (d×N)
QKᵀ              → Q 乘以 K的转置 = 注意力分数 (N×N)
√d               → d 的平方根（缩放因子，防止数值太大）
QKᵀ / √d         → 每个分数都除以 √d
softmax(...)     → 每一行变成概率分布 (N×N)
(...) V          → 结果矩阵乘以 V (N×d)
O                → 最终输出 (N×d)
```

### 公式 2：Softmax 反向传播

```
dSᵢ = Pᵢ ⊙ (dPᵢ - 1 · (PᵢᵀdPᵢ))
```

逐个拆解：

```
dSᵢ             → 损失对 S 第 i 行的梯度（向量）
Pᵢ              → softmax 输出的第 i 行（向量）
⊙               → 逐元素相乘（不是矩阵乘法）
dPᵢ             → 损失对 P 第 i 行的梯度（向量）
PᵢᵀdPᵢ          → Pᵢ 转置后和 dPᵢ 做点积 = 一个标量 = Σₖ Pₖ·dPₖ
1               → 全 1 向量
1·(PᵢᵀdPᵢ)      → 把这个标量广播成一个向量（每个位置都是同一个数）
dPᵢ - 1·(...)   → dPᵢ 的每个元素都减去那个标量
Pᵢ ⊙ (...)      → 结果再和 Pᵢ 逐元素相乘
```

### 公式 3：HBM 复杂度

```
Θ(N²d² / M)
```

```
Θ(...)           → 和 O(...) 类似，表示精确的数量级
N²               → 序列长度的平方
d²               → 维度的平方
M                → SRAM 的大小（字节数）
N²d²/M           → 整体：N 和 d 越大 → 访问量越大
                          M 越大 → 访问量越小（SRAM 大了，搬运次数少了）
```
