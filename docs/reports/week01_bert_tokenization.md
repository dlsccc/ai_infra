## 3. Key Findings
1. gpu对forward过程明显加速了。
2. 序列长度和batch增大时，显著拉高时延，且不是线性增长的，因为BERT注意力复杂度接近 `O(seq_len^2)`
3. gpu计算中，tokenization时长占比更高，因为forward变快了，cpu侧tokenizer计算没同步变快，说明后续优化时，不仅要优化模型，还要优化前处理流水线。
4. 值得一提的是，为什么gpu里tokenizer时长比cpu还长，因为这里计时不仅是计算时间，还有`v.to(device)`时间，还有`torch.cuda.synchronize()`gpu同步的时间。
